# AGENTS.md ‚Äî Tuto

## What is this

Tuto is a desktop app for learning technical concepts through an AI tutor. A lesson is a feed of short, readable cards generated by Claude ‚Äî with syntax-highlighted code, Mermaid diagrams, a per-concept Practice tab of fill-in-the-blank exercises, and a structured Notes document that builds up as you go. It is a personal tool: instead of calling an API, it spawns the local `claude` CLI (Claude Code headless mode) as its AI backend. The product plan and design rationale live in `docs/app-plan.md`.

## Stack

- Shell: Electrobun ‚Äî a Bun main process plus the OS's native WebView.
- UI: React + TypeScript, bundled by Vite; the WebView loads the build from `dist/`.
- Styling: Tailwind CSS v4 with shadcn/ui components (Base UI primitives, Hugeicons, Shiki for code, mermaid.js for diagrams).
- AI backend: the `claude` CLI, spawned per turn from the Bun process. It must be installed and authenticated on the machine.
- Persistence: plain files under the OS app-data dir ‚Äî one folder per lesson holding `lesson.json`, `notes.md`, and `notes.json`. No database.
- Lint/format: Biome.
- Package manager: pnpm (required). Bun is the app runtime, not the dependency manager. Exact-version pinning is enforced by `savePrefix: ""` in `pnpm-workspace.yaml`.

## Commands

The app is a desktop binary, not a web server. All commands run through pnpm.

| Command | What it does |
|---------|-------------|
| `pnpm start` | Build the WebView and launch the desktop app from bundled views. |
| `pnpm dev` | Same, but with Vite HMR. HMR is opt-in via `TUTO_HMR=1`; auto-detecting a running dev server is intentionally disabled, so only `pnpm dev` uses it. |
| `pnpm smoke` | Run the tutor loop against the real `claude` model end to end. Spends tokens. |
| `pnpm typecheck` | `tsc --noEmit`. |
| `pnpm lint` / `pnpm format` | Biome check / format. |
